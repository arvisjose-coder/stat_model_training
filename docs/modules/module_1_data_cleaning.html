Module 1 — Data Cleaning and Exploration Review

Version fidèle au matériel original + explication du code

1 Vue d’ensemble du module

Dans ce module, nous travaillons avec le fichier Appendix+S1+-+Lizard+data+version+1.0.csv, un jeu de données de traits biologiques de reptiles/lézards

Le message central du module est :

Même lorsqu’on connaît les statistiques, il faut d’abord apprendre à connaître ses données.

2 Théorie du Module 1

2.1 Objectifs du module

Le Module 1 a pour objectifs :

  • introduire l’analyse statistique end-to-end ;
  • donner une vue d’ensemble du processus d’analyse ;
  • montrer l’importance des données tidy ;
  • introduire des méthodes de nettoyage avancées ;
  • utiliser l’analyse exploratoire pour identifier des patterns et des relations entre variables.

2.2 Idée centrale

On ne commence pas une analyse en choisissant directement un modèle.
On commence par comprendre :

  • l’unité d’observation ;
  • la structure du tableau ;
  • les types de variables ;
  • les valeurs manquantes ;
  • les outliers ;
  • les distributions ;
  • les relations entre variables.

2.3 Analyse end-to-end

Une analyse complète suit la chaîne :

Question biologique
→ données brutes
→ nettoyage
→ exploration
→ choix des variables
→ choix du modèle
→ diagnostic
→ interprétation
→ conclusion scientifique

2.4 Données tidy

Un tableau est tidy lorsque :

Principe Exemple
Une ligne = une observation une espèce
Une colonne = une variable maximum_svl, diet, latitude
Une cellule = une valeur 61, Carnivorous, 34.16

2.5 Data wrangling

Le data wrangling comprend :

  • sélectionner les variables d’intérêt ;
  • corriger les erreurs et les typos ;
  • assurer la cohérence des données ;
  • retirer les doublons ;
  • gérer les valeurs manquantes ;
  • vérifier les formats des variables ;
  • créer de nouvelles variables si nécessaire.

2.6 Exploratory Data Analysis

L’EDA permet de :

  • comprendre les types de variables ;
  • détecter des patterns ;
  • repérer des associations ;
  • vérifier des anomalies ;
  • vérifier certaines hypothèses ;
  • guider le choix du modèle statistique.

2.7 Types de données

Type statistique Exemple Modèle possible plus tard
Continue taille, masse régression linéaire
Comptage nombre d’œufs Poisson / binomial négatif
Binaire oui/non régression logistique
Multicatégorielle activité, régime alimentaire multinomial / classification
Haute dimension expression génique, SNPs PCA, clustering, régularisation
Note

Le type R n’est pas toujours le type statistique.
largest_clutch peut être integer dans R, mais statistiquement c’est un comptage.

2.8 Feature engineering

Le feature engineering inclut :

  • transformation log ;
  • scaling ;
  • normalisation ;
  • création de nouvelles variables ;
  • réduction de dimension comme la PCA.

Ces étapes peuvent simplifier les modèles et rendre les analyses plus interprétables.

Si une cellule contient un espace, elle semble vide, mais R ne la reconnaît pas nécessairement comme valeur manquante.

Si une colonne contient des nombres mélangés à du texte, R peut lire toute la colonne comme character.

Exemple :

## Corriger les variables numériques mal lues

### Objectif

Corriger les colonnes `maximum_svl` et `hatchling_neonate_svl` si elles sont lues comme du texte.

### Pourquoi ce problème arrive ?

Si une colonne contient des nombres mélangés à du texte, R peut lire toute la colonne comme `character`.

Exemple :

| Valeur       | Problème         |
|--------------|------------------|
| `61`         | numérique        |
| `58.8`       | numérique        |
| `approx. 45` | texte + nombre   |
| `< 20`       | symbole + nombre |
| `unknown`    | texte            |

### Version 1

Cette version remplace les valeurs contenant des lettres par `NA`, puis convertit en numérique.

```{r, eval=FALSE}
dfTraits$maximum_svl <- gsub(pattern = "[a-zA-Z]", replacement = NA, x = dfTraits$maximum_svl)
dfTraits$hatchling_neonate_svl <- gsub("[a-zA-Z]", NA, dfTraits$hatchling_neonate_svl)

dfTraits[, c("maximum_svl", "hatchling_neonate_svl")] <- lapply(
  dfTraits[, c("maximum_svl", "hatchling_neonate_svl")],
  as.numeric
)
```

### Explication de la version 1

`gsub("[a-zA-Z]", NA, x)` cherche les lettres dans les cellules et remplace par `NA`.

`as.numeric()` convertit ensuite les colonnes en nombres.

`lapply()` applique la conversion à plusieurs colonnes.

### Bonification pédagogique : audit avant nettoyage

Cette version est plus fine : elle inspecte les valeurs suspectes avant de modifier les données.

### Explication du code d’audit

`str_squish()` retire les espaces inutiles.

`str_detect(..., "[A-Za-z]")` détecte les lettres.

`str_detect(..., "[<>~≈]|[-–]")` détecte des symboles ou intervalles.

`parse_number()` extrait un nombre à partir d’une chaîne de texte.

### Fonction de nettoyage bonifiée

```{r}
clean_numeric_trait <- function(x) {
  x_chr <- as.character(x)
  x_chr <- str_squish(x_chr)

  x_chr[x_chr %in% c("", " ", "NA", "N/A", "na", "NaN")] <- NA_character_

  x_num <- readr::parse_number(x_chr)

  return(x_num)
}
```

### Appliquer la fonction

```{r}
if (length(num_maybe_dirty) > 0) {

```

### Explication

`quantile()` calcule les quartiles.

`lowerQuantile` correspond à Q1.

`upperQuantile` correspond à Q3.

`iqr` est l’intervalle interquartile.

`upperThreshold <- (iqr * 3) + upperQuantile` définit un seuil supérieur conservateur.

`which()` donne les indices des lignes qui dépassent ce seuil.

`dfOutliers` contient les informations taxonomiques des outliers.

::: callout-important
